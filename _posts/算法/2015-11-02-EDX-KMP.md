1.字符串
定义:由来自字母表的字符所组成的有限序列
特性字符的种类不多，而串的长度远大于字母表包含的种类数量(比如英文文章)

2.术语
字符串相等:长度相等,且对应的字符均相同
子串(sub string):连续k个字符
前缀(prefix):前k个字符
后缀(suffix):后k个字符
空串:任何串的子串,前缀,后缀
约定:任何串都是自己的子串,前缀,后缀

3.抽象数据类型(ADT,abstract data type)
length()
charAt()
substr(i,k)
prefix(k)
suffix(k)
concat(T) (串接)
equal(T)
indexOf(P) (string S是否有子串与P完全相等)

### 蛮力匹配(brutal force)
1.蛮力匹配(brutal force)
(1)逐一尝试匹配
(2)自左向右,以字符为单位移动模式串,直到发现匹配.

2.蛮力匹配(brutal force)代码 (版本1)

```
int match(char * P, char * T){  //尾部有'\0'
	size_t n = strlen(T), i = 0;
	size_t m = strlen(P), j = 0;
	while( j < m && i < n){  //自左向右逐个比对字符, j == m的时候,匹配成功; i == n的时候,匹配完毕
		if( T[i] == P[j] ) {
			i++; j++;        //若匹配,则转到下一对字符
		} else{
			//否则,T回退,P复位	  
			i -= j;		 
			j =  0;
			//再更新
			i++;
		}
	}
	
	return i - j; //在text中匹配到的位置, 假如i-j > n-m, 失败
}
```

3.蛮力匹配(brutal force)代码 (版本2)

```
int match(char * P, char * T){
	size_t n = strlen(T), i = 0; //T[i]  与P[0]对齐
	size_t m = strlen(P), j = 0; //T[i+j]与P[j]对齐
	for(i = 0; i <n - m + 1; ++i){
		 for(j = 0; j < m; ++j){
			if( T[i+j] != P[j] )
				break; //失配, P整体右移一个字符,重新比对
		 }
	if(j >= m) break; //找到匹配字符(其实j只会==m)
	}
	
	return i;
}
```

4.蛮力匹配(brutal force)性能
(1)最好情况:只经过一轮比对，确定匹配．O(m) (m是匹配串长度,和文本串无关)
(2)最坏情况:每轮对比都不对. O(m*n)
(3)最坏情况例子: P是00001,T是0000000000...01
(4)最坏情况出现概率:
字母表越小,最坏情况出现概率越高
m越大(模式串越大),最坏情况的后果更加严重
(5)所以蛮力匹配的期望还是O(n),字母表大而且平均意义而言

### KMP算法
1.KMP算法:从记忆力到预知力
(1)蛮力为何低效:
匹配串每次匹配然后再失配,都是在某个位置断掉了,然后又开始一次重复比对
例子:
000000000000
000000001
(2)初步想法:在某个点失配,跳转存于该点的记忆值
(3)在匹配失败的那个点,我们已经知道此前比对过的字符串的信息了
(4)以上也就是所谓的__记忆力__ - "从哪里跌倒就从哪里爬起来(在哪失配就在哪从头匹配)"

2.KMP算法:查询表
(1)构造查询表next[0,m):在任一位置P[j]处失败后,将j替换为next[j]
(2)与其说是借助强大的记忆,不如说是做好充分的准备

3.KMP算法:主算法

```
//类似蛮力匹配一

int match(char * P, char * T){
	int * next = buildNext(P);      //构造next表
	int n = (int) stdlen(T), i = 0; //文本串指针 Text
	int m = (int) stdlen(P), j = 0; //模式串指针 Pattern
	
	while( j < m && i < n ){ //自左向右,逐个比对字符
		if( j < 0 || T[i] == P[j] ){ //若匹配
			i++; j++;
		} else {	//否则P右移, T不回退
			j = next[j];
		}
	}
	
	delete [] next;
	
	return i - j;
}
```
